The concept

A backend request is a flow.

Not a single function call — a chain of steps: routing, logic, data, side-effects. Fluxbase runs that chain and traces every step automatically. Hover a step to see its span.

How it works

Write logic. Deploy. Trace.

1

Write a step

A function is a step in a flow. It receives input and a ctx — database, secrets, logging, tools. No boilerplate, no imports, no configuration files.

export default defineFunction({
  name: "create_user",

  handler: async ({ input, ctx }) => {
    const [user] = await ctx.db.query({
      table: "users",
      operation: "insert",
      data: { email: input.email },
    });
    ctx.log("user created", { id: user.id });
    return { userId: user.id };
  },
});
2

Deploy the flow

Run flux deploy. Every function is bundled and live in under 30 seconds. The gateway — auth, rate limiting, routing — is wired automatically. Mark a function async: true and it becomes a background flow step: queued, traced, observable.

$ flux deploy

  Deploying 3 functions...

  ✔ create_user   → gw.fluxbase.co/create_user
  ✔ list_users    → gw.fluxbase.co/list_users
  ✔ send_welcome  (async background step)

  ✔ Deployed in 22s
3

Trace the flow

Every response has an x-request-id header. Run flux trace <id> and see the full chain: which steps ran, every database query, how long each took, what was slow, and index suggestions.

$ flux trace 4f9a3b2c

  ▸ gateway                3ms
  ▸ create_user           81ms
    ▸ db:insert(users)    14ms
  ▸ send_welcome  async →  queued

  ── total: 98ms ─────────────────

Capabilities

Everything participates in the trace.

Every flow node — request, async job, workflow step, database query, tool call — emits a span. flux trace reconstructs the complete picture from any of them.

⚡

Request flows traced

A function runs, queries the database, returns a result. Gateway, function, every db query — each emits a span. One request ID covers all of it.

🔀

Async jobs traced

Mark a function async: true. It becomes a queued background step — fire-and-forget from the caller's perspective, but still fully traced and observable.

🔗

Workflow steps traced

Chain steps into a named flow with ctx.workflow.run(). Each step sees the previous output. The entire chain, sequential or parallel, appears in one trace.

🛠️

Tool calls traced

Every external call — Stripe, Slack, a third-party API — is a named tool call. Latency, status, errors. All in the trace, automatically.

Architecture

Every layer emits trace spans.
