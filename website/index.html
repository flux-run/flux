The problem

Building backends is drowning in overhead.

Every feature you ship starts with days of infrastructure work that has nothing to do with your idea.

🔧

Config before code

Before writing a line of logic, you're setting up CORS, auth middleware, rate limiting, environment variables, and connection pooling. None of it is the product.

🕵️

Debugging is a separate job

Something's slow in production. You open three dashboards, search Datadog, grep logs, guess. You never get a clean answer — just a pile of signals to correlate manually.

🌪️

Side projects die in cloud config

You have 10 tabs open: IAM roles, VPC settings, Docker registries, Terraform state. The idea was good. The infrastructure killed it before you validated anything.

How it works

Write logic. Deploy flows. See everything.

1

Write pure logic

Your function receives input and a ctx — that's it. Query your database with ctx.db, read secrets with ctx.secrets, log with ctx.log. No boilerplate, no imports, no configuration.

export default defineFunction({
  name: "create_order",

  handler: async ({ input, ctx }) => {
    const key = await ctx.secrets.get("STRIPE_KEY");
    const [order] = await ctx.db.query({
      table: "orders", operation: "insert",
      data: { user_id: input.userId, total: input.total },
    });
    ctx.log("order created", { id: order.id });
    return { orderId: order.id };
  },
});

3

Trace every request

Every request has an x-request-id. Run flux trace <id> and see the full journey: which functions ran, every database query, how long each step took, and if anything looks wrong — N+1 patterns, slow queries, missing indexes.

$ flux trace 4f9a3b2c

  ▸ create_order          88 ms
    ▸ db:insert(orders)     14 ms
    ▸ db:select(users)      284 ms  ⚠ slow

  ── Index suggestion ────────────────────
  → CREATE INDEX ON users(id);

Everything included

The full stack, assembled.

Every layer your backend needs — already wired together, already observable.

⚡

TypeScript functions

Write logic — not boilerplate. defineFunction() is your entire API surface. Deno-based, fast cold starts, isolated per function.

🗄️

Postgres, zero config

Every project gets a hosted Postgres database. Query it with ctx.db — no connection string, no ORM, no migrations script to wire up.

🔭

Traces + N+1 detection

Every request is automatically instrumented. N+1 query patterns are caught and flagged. Slow queries get index suggestions. No SDK required.

🔑

Secrets that rotate live

Store encrypted secrets with flux secrets set. Inject at runtime with ctx.secrets.get(). Rotate without redeploying.

🛡️

Gateway, already configured

Auth, CORS, rate limiting, and request routing are handled before your function runs. You write business logic. The gateway handles the rest.

🔀

Async flows built in

Mark any function async: true and it becomes a background flow step. Fire-and-forget from your main handler — queued, traced, observable.
