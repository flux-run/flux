# Fluxbase — Machine-readable API reference
# https://fluxbase.co/docs/llm.txt
# Updated: 2026-03-10

## Overview

Fluxbase is a serverless backend platform. Developers write TypeScript functions,
connect a hosted Postgres database, and deploy globally via the `flux` CLI.

---

## CLI

Install:
  curl -fsSL https://fluxbase.co/install | bash

Commands:
  flux login                          Authenticate (opens browser)
  flux create <name> [--template t]   Scaffold project from template
  flux init --project <id>            Link directory to a project
  flux deploy                         Deploy all functions in project
  flux invoke <fn> --payload <json>   Call a deployed function
  flux logs [--tail] [--function fn]  Stream function logs
  flux trace <trace-id>               Render distributed trace as flame graph
  flux secrets set <KEY> <val>        Set a project secret
  flux secrets list                   List all secret keys
  flux project create --name <n>      Create a new project
  flux project list                   List projects
  flux dev                            Run function locally with hot reload
  flux doctor                         Check CLI config and connectivity

Templates:
  todo-api          Full CRUD REST API with Postgres (create/list/update/delete)
  webhook-worker    Webhook receiver + event storage
  ai-backend        LLM proxy with logging and caching

## Functions

A function is a TypeScript module exported via defineFunction().

```typescript
import { defineFunction } from "@fluxbase/functions";

export default defineFunction({
  name: "my_function",
  handler: async ({ input, ctx }) => {
    ctx.log("invoked", input);
    return { ok: true };
  },
});
```

### FluxContext (ctx)

ctx.log(...args)             Structured log emitted with request_id
ctx.env.<KEY>                Environment variable / project secret
ctx.secrets.get("KEY")       Retrieve encrypted secret (async)
ctx.db.query(params)         Execute a database query (see below)

### flux.json (per function)

{
  "name": "function_name",
  "runtime": "deno",          // "deno" | "node"
  "route": "/function_name",  // HTTP path (default: /name)
  "method": "POST",           // HTTP method (default: POST)
  "async": false              // true = fire-and-forget
}

## Database

ctx.db.query() params:

  table      string   required   Table name
  operation  string   required   "select" | "insert" | "update" | "delete"
  filters    array    optional   [{column, op, value}]
  columns    array    optional   Columns to return (SELECT)
  data       object   optional   Row data (INSERT / UPDATE)
  limit      number   optional   Max rows
  offset     number   optional   Pagination offset

Filter operators: eq neq gt lt gte lte like in is_null

Examples:

  // SELECT
  const rows = await ctx.db.query({
    table: "todos",
    operation: "select",
    filters: [{ column: "done", op: "eq", value: false }],
    limit: 50,
  });

  // INSERT
  const [row] = await ctx.db.query({
    table: "todos",
    operation: "insert",
    data: { title: "Buy milk", done: false },
  });

  // UPDATE
  await ctx.db.query({
    table: "todos",
    operation: "update",
    filters: [{ column: "id", op: "eq", value: id }],
    data: { done: true },
  });

  // DELETE
  await ctx.db.query({
    table: "todos",
    operation: "delete",
    filters: [{ column: "id", op: "eq", value: id }],
  });

## API Gateway

Base URL: https://gw.fluxbase.co
Auth header: X-API-Key: <project-api-key>

  POST /function_name        Invoke function (synchronous)
  x-request-id header        Trace ID — pass to `flux trace`

---

## REST API (management)

Base URL: https://api.fluxbase.co
Auth header: Authorization: Bearer <firebase-jwt>

  GET  /tenants                         List tenants
  GET  /projects                        List projects
  POST /projects                        Create project
  GET  /schema/graph                    Schema graph for a project
  GET  /logs?limit=N                    Function logs
  GET  /logs/trace?trace_id=<id>        Distributed trace

Trace response shape:
  {
    "spans": [...],
    "n1_warnings": [{"table", "count", "message"}],
    "suggested_indexes": [{"table", "column", "ddl"}]
  }

---

## Observability

All requests are auto-instrumented. No configuration required.

Span types:
  function        Total function execution
  db:query        Per-query span: table, operation, duration_ms, cache (HIT|MISS|BYPASS)

Features:
  N+1 detection       Warns when same table is queried >5 times in one request
  Slow query alert    Flags queries >100 ms
  Index suggestions   Generates CREATE INDEX DDL for repeatedly-slow filter columns

## Errors

Functions return HTTP 200 on success. Errors return HTTP 4xx/5xx with:
  { "error": "message" }

Common gateway errors:
  401 Unauthorized — missing or invalid X-API-Key
  429 Too Many Requests — rate limit exceeded
  500 Internal Server Error — function threw an uncaught exception
