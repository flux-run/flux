Deploy your first backend in 5 minutes

From a fresh machine to a live API — using the Fluxbase CLI.

  1. 1

    Install the CLI

    One command — detects your OS and architecture automatically.

  2. 2

    Log in

    Saves a credential token locally.

  3. 3

    Scaffold a project

    Pick a template — 4 CRUD functions and a Postgres schema are generated for you.

  4. 4

    Deploy

    All functions in the project are bundled and deployed in a single command.

  5. 5

    Call the API

    Your functions are live at a public endpoint. Call them with curl or any HTTP client.

Full walkthrough

Step 1 — Install

$ curl -fsSL https://fluxbase.co/install | bash
# Fluxbase CLI 0.1.0 installed to /usr/local/bin/flux

$ flux --version
flux 0.1.0

Step 2 — Log in

$ flux login
# Opens browser → sign in with Google
# → Logged in as you@example.com

Step 3 — Create a project

Create a new project from the todo-api template. The interactive picker lets you choose, or pass --template directly:

$ flux create my-todo-api --template todo-api

  flux create

  Scaffolding todo-api template into my-todo-api/ ...

    + my-todo-api/functions/create_todo/index.ts
    + my-todo-api/functions/list_todos/index.ts
    + my-todo-api/functions/update_todo/index.ts
    + my-todo-api/functions/delete_todo/index.ts
    + my-todo-api/schema/todos.sql
    + my-todo-api/README.md
    + my-todo-api/.env.example
    + my-todo-api/.gitignore
  ✔ Project created: my-todo-api/
$ cd my-todo-api

Step 4 — Link to a Fluxbase project

# Create a project in Fluxbase if you don't have one yet
$ flux project create --name "My Todo App"
# → project_id: 3787e1fa-...-bccc12

# Link this directory to that project
$ flux init --project 3787e1fa-...-bccc12

Step 5 — Deploy

$ flux deploy

  Deploying 4 functions...

  ✔ create_todo   → https://gw.fluxbase.co/create_todo
  ✔ list_todos    → https://gw.fluxbase.co/list_todos
  ✔ update_todo   → https://gw.fluxbase.co/update_todo
  ✔ delete_todo   → https://gw.fluxbase.co/delete_todo

  Deployed in 8.2s

Step 6 — Call the API

# Create a todo
$ curl -X POST https://gw.fluxbase.co/create_todo \
    -H "Content-Type: application/json" \
    -H "X-API-Key: YOUR_API_KEY" \
    -d '{"title": "Launch on Fluxbase"}'
{"id":"c7f2...","title":"Launch on Fluxbase","done":false}

# List all todos
$ curl https://gw.fluxbase.co/list_todos \
    -H "X-API-Key: YOUR_API_KEY"
{"todos":[{"id":"c7f2...","title":"Launch on Fluxbase","done":false}]}

What got deployed

Each function in functions/ is an independent serverless unit:

my-todo-api/
  functions/
    create_todo/
      index.ts    ← your function handler
      flux.json   ← name, runtime, route config
      package.json
    list_todos/   ...
    update_todo/  ...
    delete_todo/  ...
  schema/
    todos.sql     ← Postgres DDL
  .env.example    ← required env vars
  README.md

Open functions/create_todo/index.ts to see the function code:

import { defineFunction } from "@fluxbase/functions";

export default defineFunction({
  name: "create_todo",
  handler: async ({ input, ctx }) => {
    const [todo] = await ctx.db
      .query({ table: "todos", operation: "insert",
               data: { title: input.title, done: false } });
    return todo;
  },
});

Inspect traces

After invoking your functions, view the distributed trace:

$ flux logs --tail
$ flux trace <trace-id>
