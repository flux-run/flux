What is Fluxbase?

Fluxbase is a backend flow runtime. You write TypeScript functions — each one a step in a flow. Fluxbase handles everything else: routing requests through the gateway, running your functions in an isolated Deno runtime, executing database queries through the data engine, managing secrets, queuing async work, and tracing every request end-to-end.

The defining feature is observability. Every request gets a unique ID. Every hop — gateway auth check, function execution, database query, async queue dispatch — is recorded as a span. Run flux trace <id> and see the full picture.
