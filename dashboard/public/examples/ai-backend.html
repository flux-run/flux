AI Backend

Proxy requests to OpenAI with built-in logging, rate limiting, and response persistence — deployed as a single Fluxbase function.

Deploy in one command

$ flux create my-ai --template ai-backend
$ cd my-ai
$ flux secrets set OPENAI_API_KEY "sk-..."
$ flux init --project <PROJECT_ID>
$ flux deploy

Database schema

CREATE TABLE IF NOT EXISTS ai_queries (
  id           UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  prompt       TEXT NOT NULL,
  model        TEXT NOT NULL DEFAULT 'gpt-4o-mini',
  response     TEXT,
  tokens_used  INT,
  latency_ms   INT,
  created_at   TIMESTAMPTZ DEFAULT now()
);

Function

ai_query — POST /ai_query

// functions/ai_query/index.ts
import { defineFunction } from "@fluxbase/functions";

export default defineFunction({
  name: "ai_query",
  handler: async ({ input, ctx }) => {
    const model  = (input.model as string) ?? "gpt-4o-mini";
    const prompt = input.prompt as string;

    // Retrieve API key from secrets
    const apiKey = ctx.env.OPENAI_API_KEY;
    if (!apiKey) throw new Error("OPENAI_API_KEY not configured");

    ctx.log("querying LLM", { model, prompt_len: prompt.length });

    const start = Date.now();

    // Call OpenAI
    const res = await fetch("https://api.openai.com/v1/chat/completions", {
      method: "POST",
      headers: {
        "Authorization": `Bearer ${apiKey}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        model,
        messages: [{ role: "user", content: prompt }],
      }),
    });

    if (!res.ok) throw new Error(`OpenAI error: ${res.status}`);

    const data       = await res.json();
    const answer     = data.choices[0]?.message?.content ?? "";
    const tokensUsed = data.usage?.total_tokens ?? 0;
    const latency    = Date.now() - start;

    // Persist to database
    await ctx.db.query({
      table: "ai_queries", operation: "insert",
      data: { prompt, model, response: answer, tokens_used: tokensUsed, latency_ms: latency },
    });

    ctx.log("LLM response", { tokens: tokensUsed, latency_ms: latency });

    return { answer, model, tokens_used: tokensUsed, latency_ms: latency };
  },
});

Usage

$ curl -X POST https://gw.fluxbase.co/ai_query \
    -H "X-API-Key: $API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"prompt":"What is Fluxbase?","model":"gpt-4o-mini"}'

{
  "answer": "Fluxbase is a serverless backend platform...",
  "model": "gpt-4o-mini",
  "tokens_used": 142,
  "latency_ms": 834
}

Query stored responses

# List recent queries from the database
$ curl -X POST https://gw.fluxbase.co/list_queries \
    -H "X-API-Key: $API_KEY" \
    -d '{"limit":10}'

Extend this example

  • Add a cache layer — check ai_queries for identical prompts before calling OpenAI
  • Add streaming — return a stream response for real-time output
  • Add cost tracking — log token costs per user or project
  • Add model routing — route cheap prompts to smaller models automatically
Observability included
Every call is traced automatically. View latency, token counts, and slow queries with flux trace <request-id>.
