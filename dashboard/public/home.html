The concept

A backend request is a flow.

Not a single function call — a chain of steps: routing, logic, data, side-effects. Fluxbase runs that chain and traces every step automatically. Hover a step to see its span.

How it works

Write logic. Deploy. Trace.

1

Write a step

A function is a step in a flow. It receives input and a ctx — database, secrets, logging, tools. No boilerplate, no imports, no configuration files.

export default defineFunction({
  name: "create_user",

  handler: async ({ input, ctx }) => {
    const [user] = await ctx.db.query({
      table: "users",
      operation: "insert",
      data: { email: input.email },
    });
    ctx.log("user created", { id: user.id });
    return { userId: user.id };
  },
});
2

Deploy the flow

Run flux deploy. Every function is bundled and live in under 30 seconds. The gateway — auth, rate limiting, routing — is wired automatically. Mark a function async: true and it becomes a background flow step: queued, traced, observable.

$ flux deploy

  Deploying 3 functions...

  ✔ create_user   → gw.fluxbase.co/create_user
  ✔ list_users    → gw.fluxbase.co/list_users
  ✔ send_welcome  (async background step)

  ✔ Deployed in 22s
3

Trace the flow

Every response has an x-request-id header. Run flux trace <id> and see the full chain: which steps ran, every database query, how long each took, what was slow, and index suggestions.

Architecture

Every layer emits trace spans.

flux trace reconstructs the entire flow by replaying every span in order — gateway to function to database to tools. You don't configure tracing. It's built into the runtime.

Capabilities

What you can build

Fluxbase runs four kinds of flows. All traced the same way.

⚡

Request flows

Standard request → response. A function runs, queries the database, returns a result. Every step is traced, every query timed.

🔀

Async background flows

Mark a function async: true. It becomes a queued background step — fire-and-forget, but still fully traced and observable.

🔗

Workflow flows

Chain steps into a named flow with ctx.workflow.run(). Each step sees the previous output. Sequential or parallel.

🤖

Agent flows

Give an LLM access to your tools via ctx.agent.run(). It decides which tool to call next, loop by loop. Every LLM call and tool execution is traced.
