CLI Reference

Debug production
in one command.

Fluxbase records every backend execution. The CLI lets you inspect production requests like Git commits — right from your terminal.

flux why 550e8400 
$ flux why 550e8400

  ✗  POST /signup → create_user  (3200ms, 500)

  ─── Root cause ──────────────────────────────────────────
  Stripe timeout after 10000ms

  ─── Execution graph ─────────────────────────────────────
  gateway     POST /signup           2ms
  runtime     create_user            1ms
  db          users (SELECT)        12ms
  tool        stripe.charge       3200ms  ⚠ slow

  ─── State changes ───────────────────────────────────────
  users  INSERT  email=user@example.com  plan=free

  ─── Suggested next steps ────────────────────────────────
  flux doctor 550e8400        full diagnosis
  flux trace diff 550e8400 4f9a3b2c  compare traces

Core Workflow

Three commands. Every incident resolved.

The entire debugging workflow — from spotting a failure to understanding its cause — is three commands.

WATCH
flux tail

Stream live requests

flux tail 
$ flux tail

  METHOD   ROUTE          DURATION  STATUS
  POST     /login         38ms     ✔
  POST     /checkout      121ms    ✔
  POST     /signup        3.2s     ✗ 500
    Stripe timeout after 10000ms
    → flux why 550e8400
UNDERSTAND
flux why

Root-cause a failed request

flux why 550e8400 
$ flux why 550e8400

  ✗  POST /signup  3200ms  500

  Root cause: Stripe timeout after 10000ms

  gateway   2ms
  runtime   1ms
  db        12ms
  stripe    3200ms  ⚠
DIAGNOSE
flux doctor

Automatic incident diagnosis

flux doctor 550e8400 
$ flux doctor 550e8400

  ROOT CAUSE
  ⚡ stripe.charge timed out after 10000ms

  LIKELY ISSUE
  External tool latency exceeded threshold

  SUGGESTED ACTIONS
  • Increase timeout above 11000ms
  • Add retry with exponential backoff
Deep Debugging

Superpowers that no other tool has.

Traditional APM shows you that a request failed. Fluxbase lets you step through it, diff it, replay it, and bisect it — all from the CLI.

🔬
Step through a request
flux trace debug 550e8400

Navigate spans with arrow keys. See exact inputs, outputs, and timing for every step of execution.

flux trace debug 
Step 1/4  gateway
  ──────────────────────────────────
  Input:  POST /signup  { email: "a@b.com" }
  Output: { passed: true }
  Time:   4ms

  ↓ next  ↑ prev  q quit
⚖️
Compare two executions
flux trace diff reqA reqB

Diff any two request traces side by side. Changed spans highlighted — useful for before/after a deploy.

flux trace diff 
  SPAN              A        B        DELTA
  gateway           3ms     4ms     +1ms
  create_user      81ms    44ms    -37ms
  stripe.charge    12ms  10002ms  +9990ms ✗

  → stripe.charge regressed
⏪
Replay a production incident
flux incident replay 14:00..14:05

Re-execute any time window against your current code. Side-effects off. Database writes on.

flux incident replay 
  ✔  4f9a3b2c  POST /create_user   200
  ✔  a3c91ef0  GET  /list_users    200
  ✗  550e8400  POST /signup        500
     └─ Still failing: Stripe timeout

  23 replayed · 1 still failing
🔎
Find the bad commit
flux bug bisect --request 550e8400

Binary-searches your git history. Finds the exact commit and author that introduced the regression.

flux bug bisect 
  Testing abc123…  ✔ passes
  Testing def456…  ✗ fails

  FIRST BAD COMMIT
  def456  "feat: add retry to stripe.charge"
  2026-03-08  alice@example.com
Mental Model

If Git is version history for code, Fluxbase is version history for production.

The same mental model. The same commands. Applied to your production execution graph instead of your source tree.

Git Fluxbase What it answers
git log flux tail See what happened
git show flux why Inspect one execution
git diff flux trace diff Compare two executions
git bisect flux bug bisect Find the bad commit
git blame flux state blame Who changed this row?
Full Command Surface

Every command answers a developer question.

The CLI has 11 commands. Each one maps to a question you'd ask when something goes wrong in production.

Command What it answers
flux tail What is happening right now?
flux why <id> Why did this request fail?
flux doctor <id> What is the likely cause?
flux trace debug <id> What happened step-by-step?
flux trace diff <a> <b> What changed between two runs?
flux state history <table> --id <row> How did this row change over time?
flux state blame <table> --id <row> Who modified this field last?
flux incident replay <from>..<to> What happens if we replay this window?
flux bug bisect --request <id> Which deploy introduced the bug?
flux explain <query-file> What SQL will run? What policies apply?
flux deploy Deploy functions to production
30-Second Workflow

Deploy. Tail. Why. Fixed.

The core debugging workflow: spot a failure in flux tail, take the request ID, hand it to flux why. Everything else is optional depth.

debug in 30 seconds 
# 1. Deploy
$ flux deploy
  ✔  create_user → gw.fluxbase.co/create_user

# 2. Watch for failures
$ flux tail
  POST /signup  3.2s  ✗ 500
  Stripe timeout after 10000ms
  → flux why 550e8400

# 3. Root-cause immediately
$ flux why 550e8400
  Root cause: Stripe timeout after 10000ms
  stripe.charge  3200ms  ⚠ slow

# 4. Auto-diagnose
$ flux doctor 550e8400
  • Increase timeout above 11000ms
  • Add retry with exponential backoff
  1. 1

    Deploy your function

    flux deploy bundles and ships your TypeScript in ~20s. Returns a URL per function.

  2. 2

    Watch for failures

    flux tail streams live requests. Errors appear in red with their request ID inline.

  3. 3

    Root-cause immediately

    flux why <id> shows root cause, execution graph, and state changes in one output.

  4. 4

    Go deeper if needed

    flux doctor, flux trace diff, flux incident replay, flux bug bisect — each adds a layer of precision.

Deploy & Runtime
flux deploy

Deploy functions to production

Bundles all functions in the current project, uploads them, and makes them live behind the gateway in ~20 seconds. Returns a deploy ID and the public URL for each function.

flux deploy 
$ flux deploy

  Deploying 3 functions…

  ✔  create_user   → gw.fluxbase.co/create_user
  ✔  list_users    → gw.fluxbase.co/list_users
  ✔  send_welcome  (async)

  ✔  Deployed in 18s  deploy:d_7f3a9
flux tail

Stream live request logs

Real-time request stream with inline data mutations and error messages. Every request shows method, path, duration, and the rows it changed. Errors show the reason immediately.

flux tail 
$ flux tail

   cart_items.id=a2f1  insert

   error: Stripe timeout after 10000ms
   → flux why 550e8400

   users.id=7f3a  plan free → pro

Incident Analysis
flux bug bisect

Find the commit that introduced a bug

Binary-searches your git history comparing trace behaviour before/after each commit. Automatically identifies the first commit where a specified request started failing.

flux bug bisect 
$ flux bug bisect --request 550e8400

  Bisecting 42 commits (2026-03-01..2026-03-10)…

  Testing commit abc123…  ✔ passes
  Testing commit def456…  ✗ fails

  FIRST BAD COMMIT
  def456  "feat: add retry logic to stripe.charge"
  2026-03-08 by alice@example.com
